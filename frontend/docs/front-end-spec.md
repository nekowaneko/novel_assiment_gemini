# AI 寫作伴侶 - 前端重構 UI/UX 規格書 (UI/UX Specification)

## 1. 介紹 (Introduction)

本文件定義了 **AI 寫作伴侶** 新版使用者介面的使用者體驗目標、資訊架構、使用者流程與視覺設計規範。它將作為視覺設計與前端開發的基礎，確保我們能打造出一致且以使用者為中心的體驗。

### 目的
本專案旨在將現有的 Streamlit 應用程式遷移至以 **Next.js** 為基礎的架構。核心目標是引入專業級的 **富文本編輯器 (Rich Text Editor)**，提供更流暢、無干擾且具備「沉浸感」的寫作體驗，同時保留並優化與 AI (Gemini) 的互動流程。

### 範圍
-   **核心平台**：Next.js 單頁應用程式 (SPA)。
-   **編輯器**：整合現代富文本函式庫 (如 Tiptap, Lexical)，支援即時格式化 (粗體、斜體等) 且無 Markdown 語法干擾。
-   **AI 整合**：
    -   **側邊欄 RAG 聊天室**：可持續存取知識庫與編劇 AI。
    -   **行內輔助 (Inline Assistance)**：在編輯器內直接提供具有上下文感知的 AI 建議。
-   **資料流**：與現有的 Python 後端 API 進行通訊，處理檔案操作與 AI 推論。
-   **語言**：介面與文件均採用繁體中文 (Traditional Chinese)。

## 2. UX 目標與原則 (UX Goals & Principles)

### 目標使用者畫像 (Target User Personas)
*   **文心 (獨立小說創作者)**
    *   **背景**：28歲上班族，利用業餘時間寫作。
    *   **核心需求**：極度討厭被打斷，需要一個能快速進入並維持「心流 (Flow)」的環境。
    *   **痛點**：舊版介面切換視窗繁瑣，且無法直接在編輯時預覽粗體/斜體等格式效果。

### 易用性目標 (Usability Goals)
1.  **沉浸感 (Immersion)**：介面應盡可能「隱形」，編輯模式下應隱藏不必要的選單與狀態列，讓使用者專注於文字本身。
2.  **即時性 (Responsiveness)**：打字、選取文字、呼叫 AI 面板的視覺回饋延遲應極低，確保操作的流暢感。
3.  **可控性 (Controllability)**：使用者必須對 AI 的介入有完全的控制權，AI 不應自動覆蓋或修改使用者的文字。

### 核心設計原則 (Design Principles)
1.  **內容至上 (Content First)**：螢幕空間應優先留給寫作畫布。
2.  **所見即所得 (WYSIWYG)**：摒棄 Markdown 語法標記 (如 `**text**`) 的直接顯示，轉而提供直覺的富文本樣式渲染。
3.  **輔助而不干擾 (Assist, Don't Intrude)**：AI 是顧問而非代筆者，其建議應以「註解」或「側邊對話」呈現，而非直接插入正文。

## 3. AI 互動設計 (AI Interaction Design)

根據使用者需求，AI 互動將遵循以下模式：

### A. 啟動方式 (Trigger)
*   **VS Code 風格側邊欄**：
    *   介面右上方或右側邊緣設有明顯的「編劇AI」與「讀者AI」圖示按鈕。
    *   點擊按鈕可**切換 (Toggle)** 右側聊天室面板的顯示與隱藏。
    *   當面板隱藏時，編輯器佔據最大寬度；當面板展開時，編輯器自動調整寬度 (Responsive)。

### B. 回覆呈現 (Presentation)
*   **側邊註解 (Side Comments)**：
    *   針對特定段落的 AI 建議 (如：潤飾、擴寫) **不會**直接插入編輯器游標處。
    *   回覆將顯示在右側面板中，或以類似 Google Docs/Word 的「註解卡片」形式顯示在編輯區邊緣。
    *   使用者需手動點擊「採納」或複製文字，才會更動正文內容。

### C. 上下文管理 (Context Awareness)
*   **手動知識庫選擇 (Manual Selection)**：
    *   延續原後端設計邏輯。
    *   在 AI 側邊欄中，提供「選擇知識庫/文件」的介面 (如下拉選單或勾選框)。
    *   使用者必須主動指定 AI 當前對話應參考哪些設定檔 (世界觀、角色等)。

## 4. 資訊架構 (Information Architecture)

### Sitemap / Screen Inventory

本應用為單頁應用程式 (SPA)，核心畫面為「編輯工作台 (Editor Workbench)」。

1.  **主畫面 (Main Layout)**
    *   **預設狀態 (Default State)**：僅顯示中央編輯區與頂部簡潔的工具列/導航列，左側知識庫與右側 AI 面板預設為 **隱藏/摺疊**。
    *   **頂部導航列 (Top Bar)**：
        *   **左側**：檔案名稱、字數、左側邊欄開關。
        *   **右側**：匯出/設定、**編劇AI 開關**、**讀者AI 開關**。
        *   **下方 (Tabs)**：類似 VS Code 的分頁標籤列，顯示目前開啟的文件 (如「第一章」、「世界觀設定」)。
    *   **左側邊欄 (Knowledge Sidebar)**：
        *   點擊頂部開關後展開。
        *   內容：專案檔案樹狀圖 (小說章節、知識庫文件)、上傳/更新按鈕。
    *   **右側邊欄 (AI Assistant)**：
        *   點擊頂部對應按鈕後展開。
        *   內容：AI 聊天介面、上下文選擇器。
    *   **中央編輯區 (Editor Canvas)**：
        *   顯示當前活躍分頁 (Active Tab) 的內容。
        *   **可調整寬度 (Resizable)**：支援拖曳邊緣調整閱讀寬度。
        *   **視覺模擬**：中央區塊呈現類似 A4 紙張的視覺效果 (陰影、圓角、頁邊距)。

## 5. 使用者流程 (User Flows)

### Flow 1: 多分頁編輯與知識庫維護 (Tab-based Editing & KB Update)
**目標**：讓使用者在不離開編輯環境的情況下，像切換程式碼檔案一樣切換「小說正文」與「設定集」，並進行修改更新。

1.  **開啟文件**：
    *   使用者點擊左側邊欄的「世界觀設定」。
    *   中央編輯區上方新增一個名為「世界觀設定」的 **分頁 (Tab)**。
    *   畫面自動切換顯示該分頁內容，使用者可直接進行編輯。
2.  **切換編輯**：
    *   使用者點擊上方的「第一章.docx」分頁標籤，畫面瞬間切換回小說正文。
    *   使用者點擊「世界觀設定」分頁標籤，畫面切換回設定集。
3.  **儲存與更新**：
    *   使用者在「世界觀設定」分頁中修訂內容後。
    *   點擊介面上的 **「儲存並更新知識庫」** 按鈕 (或使用快捷鍵 `Ctrl+S`)。
    *   **系統回饋**：顯示「正在同步AI大腦...」進度條。
    *   後端完成向量重建後，提示「更新完成」，此時 AI 已具備最新知識。

### Flow 2: 雙 AI 協作模式 (Dual AI Collaboration)
**目標**：區分「編劇」與「讀者」的不同上下文需求。

#### 2a. 編劇 AI (Screenwriter AI) - 宏觀策劃
1.  **情境**：作者卡關，需要討論劇情發展。
2.  **觸發**：點擊右上角「編劇AI」圖示，右側面板展開。
3.  **上下文設定**：
    *   面板上方顯示 **「知識庫選擇器」**。
    *   使用者勾選相關的設定檔 (如「人物小傳」、「魔法規則」)。
    *   AI 讀取這些設定作為背景知識 (System Prompt / RAG)。
4.  **互動**：使用者提問，AI 根據設定回答建議。

#### 2b. 讀者 AI (Reader AI) - 微觀審閱
1.  **情境**：作者寫完一段，想知道讀者觀感。
2.  **觸發**：點擊右上角「讀者AI」圖示 (或選取文字後浮現的「讀者評價」)。
3.  **上下文設定**：
    *   **模式 A (選取)**：使用者在編輯器中反白一段文字 -> AI 僅針對這段文字評價。
    *   **模式 B (章節)**：使用者在 AI 面板選擇「第一章」(或多章) -> AI 讀取整章內容進行分析。
4.  **互動**：AI 給出情緒分析、節奏評論或改進建議。

## 6. 視覺設計 (Visual Design)

### 風格 (Style)
**傳統文書處理風格 (Traditional Word Processor Style)**
目標是提供使用者熟悉的寫作環境，模擬在 Microsoft Word 或紙本上寫作的感覺。

### 組件庫 (Component Library)
*   **MUI (Material UI)**：使用其豐富且成熟的組件系統，提供穩定的操作體驗。

### 色彩 (Color)
*   **背景**：
    *   **編輯器畫布**：純白 (#FFFFFF) 或 極淺灰 (#FAFAFA)，模擬 A4 紙張質感。
    *   **應用程式背景**：深灰色 (#F0F2F5) 或 桌面材質色，用以襯托中央的「紙張」。
*   **主色 (Primary)**：
    *   深藍色 (如 Word 的標題列風格) 或 經典黑。
*   **文字**：
    *   高對比黑色 (#000000) 用於正文。

### 排版 (Typography)
*   **字體**：
    *   **標題與介面**：使用系統預設襯線體或無襯線體。
    *   **正文 (小說明體)**：優先使用 **標楷體 (KaiTi)** 或 **新細明體 (PMingLiU)**，若無則退回到 **Noto Serif TC (思源宋體)**，確保傳統中文小說的閱讀質感。
    *   **西文**：Times New Roman。
*   **排版細節**：
    *   **行距**：1.5 倍行高。
    *   **段落**：首行縮排 2 字元 (全形)。
    *   **邊距**：模擬 A4 紙張頁邊距 (上下左右 2.54cm)。
