# 技術堆疊文件 (Tech Stack Document)
# 專案名稱： AI 寫作伴侶 (AI Writing Companion)

## 1. 專案概述 (Project Overview)
本文件詳述「AI 寫作伴侶」專案所採用的技術堆疊。我們的選擇旨在實現最短的開發時程與最高的開發效率，同時確保核心功能的穩定運行。此技術棧完全基於 Python 生態系，無需複雜的前後端分離架構，專為個人獨立開發者量身打造。

## 2. 核心技術棧 (Core Tech Stack)		

| 類別 | 技術名稱 | 詳細說明與選擇原因 |
| :--- | :--- | :--- |
| **應用框架** | Streamlit | 作為單體應用程式的核心框架。它允許我們僅使用 Python 程式碼來建立完整且互動性高的網頁使用者介面。這消除了對 HTML/CSS/JavaScript 的需求，大幅縮短了 UI 開發時間。 |
| **LLM 邏輯** | LangChain | 一個專門用於開發 LLM 應用程式的框架。我們將利用它的記憶體 (Memory) 功能，來高效地管理與「編劇 AI」進行多輪對話時的上下文 (context)。這能避免從零開始手動處理複雜的對話歷史，加速功能開發。 |
| **LLM 服務** | Google AI SDK | Google 官方提供的 Python 函式庫，用於與 Gemini API 進行通訊。它提供了穩定且經過最佳化的方式來發送請求並處理回覆，確保 LLM 相關功能的穩定性與效能。 |
| **文件處理** | python-docx | 一個用於處理 .docx 檔案的熱門 Python 函式庫。它將用於讀取使用者上傳的小說內容，並將編輯後的內容寫回檔案，實現檔案在本地端的存取與修改。 |
| **檔案儲存** | 本地端檔案系統 | 為了符合產品對於資料隱私的核心要求，所有使用者上傳的 .docx 檔案都將直接儲存在其本機電腦上。應用程式會透過 Python 標準函式庫進行檔案讀寫，不涉及任何雲端伺服器。 |

## 3. 依賴與工具 (Dependencies & Tools)

為了確保專案環境的可複製性，所有必要的 Python 套件都將列在 requirements.txt 檔案中。

requirements.txt 內容：

*  streamlit
*  langchain
*  google-generativeai
*  python-docx

## 4. 選擇此技術棧的優勢 (Advantages of This Stack)

此技術堆疊的結合為專案帶來了以下關鍵優勢：

*  極速開發： 從 UI 到 LLM 邏輯都統一使用 Python，顯著減少了技術堆棧的複雜性，讓您可以專注於核心功能的實現。

*  單一語言開發： 避免了前後端語言切換的困擾，讓開發流程更流暢，並能有效減少常見的程式錯誤。

*  高維護性： 儘管是單體架構，但透過 LangChain 的模組化設計，LLM 相關的複雜邏輯將能保持清晰與有條理，方便未來的維護與擴充。

*  符合核心需求： 完美滿足了「快速原型開發」、「獨立開發者友好」與「資料隱私」等所有核心產品要求。
