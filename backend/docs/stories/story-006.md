# Story: 多重對話與狀態管理

## Story (使用者故事)
作為一名小說作者，我希望能在應用程式中管理多個獨立的對話視窗，並能隨時切換、重命名或刪除這些對話，以便我在處理不同的小說主題（如角色背景、世界觀、情節發想）時能保持思路清晰。

## Acceptance Criteria (驗收標準)
為了確保此功能已完整實現並符合預期，開發者需滿足以下條件：

*   **SCENARIO 1: 開啟新對話**
    *   **Given** 使用者正在與 AI 聊天
    *   **When** 使用者點擊「開啟新對話」按鈕
    *   **Then** 應用程式應清空當前對話介面，並在側邊欄生成一個新的對話紀錄，確保新的對話不會帶入舊有的上下文。

*   **SCENARIO 2: 切換對話歷史**
    *   **Given** 使用者擁有多個歷史對話紀錄
    *   **When** 使用者在側邊欄點擊其中一個對話標題
    *   **Then** 應用程式應正確載入該對話的所有歷史訊息與相關聯的知識庫選取狀態。

*   **SCENARIO 3: 對話管理（重命名與刪除）**
    *   **Given** 使用者在側邊欄檢視對話列表
    *   **When** 使用者對特定對話選擇「重命名」或「刪除」
    *   **Then** 應用程式應即時更新側邊欄列表，並在刪除時提供二次確認以防誤刪。

*   **SCENARIO 4: 狀態持久化（選配）**
    *   **Given** 使用者關閉並重新開啟應用程式
    *   **When** 應用程式啟動時
    *   **Then** 先前的對話歷史應能被重新載入（可儲存於本地 JSON 或資料庫中）。
