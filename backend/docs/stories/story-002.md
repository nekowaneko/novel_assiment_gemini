# Story: 

## Story (使用者故事)
作為一名小說作者，我希望能選取小說中的一個片段，並呼叫讀者 AI 給出讀後評價，以便我能從讀者的角度審視我的文字，並獲得新的修改靈感。

## Acceptance Criteria (驗收標準)
為了確保此功能已完整實現並符合預期，開發者需滿足以下條件：

*   **SCENARIO 1: 選取並呼叫**
    *   **Given** 應用程式已載入小說內容，並且使用者已選取了一個文字片段
    *   **When** 使用者點擊一個明確的按鈕或觸發一個操作（例如：右鍵選單）來呼叫「讀者 AI」
    *   **Then** 應用程式會將選取的文字片段作為輸入，傳送給 LLM 服務。

*   **SCENARIO 2: 處理 AI 回應** 
    *  **Given** 應用程式已成功將片段傳送給 LLM 服務
    *  **When** LLM 服務返回讀後評價
    *  **Then** 應用程式能將 AI 的回覆顯示在一個專用的介面區域（例如：側邊欄或彈出視窗），並清晰地標示為「讀者 AI 的評價」。

*   **SCENARIO 3: 錯誤處理**
    *  **Given** 使用者嘗試呼叫讀者 AI
    *  **When** 發生 API 錯誤（例如：網路中斷或金鑰無效）
    *  **Then** 應用程式應顯示一個友善的錯誤訊息，告知使用者無法完成請求。
