# Story: 

## Story (使用者故事)
作為一名小說作者，我希望在與編劇 AI 討論時，能自動引用我上傳的小說、世界觀或人物設定等文件中的相關內容，以便在不手動複製貼上的情況下，讓 AI 擁有更豐富的上下文，提供更精準且有創意的建議。

## Acceptance Criteria (驗收標準)
為了確保此功能已完整實現並符合預期，開發者需滿足以下條件：

*   **SCENARIO 1: 文件分類與標記**
    *   **Given** 使用者上傳了一個 .docx 檔案
    *   **When** 使用者選擇將此檔案標記為「小說內容」、「世界觀」或「人物設定」
    *   **Then** 應用程式能將該檔案的內容儲存於對應的「知識庫」中，並做好準備以供後續檢索。

*   **SCENARIO 2: 智慧型檢索與上下文引用** 
    *  **Given** 使用者已將文件標記為「世界觀」與「人物設定」並上傳
    *  **And** 使用者在聊天輸入框中提問關於特定情節或角色的問題
    *  **When** 應用程式將使用者的問題傳送給後端
    *  **Then** 後端能自動檢索「世界觀」與「人物設定」知識庫中與該問題最相關的片段，並將這些片段作為上下文，與使用者的問題一起傳送給 Gemini。

*   **SCENARIO 3: 回應的精準度**
    *  **Given** 應用程式已成功將問題與檢索到的上下文傳送給 Gemini
    *  **When** Gemini 返回回覆
    *  **Then** AI 的回覆應明確參考或整合檢索到的上下文內容，顯示出對使用者作品的深度理解。

*   **SCENARIO 4: 處理 Token 限制**
    *  **Given** 檢索到的相關內容總量超過了 Gemini 模型所能處理的 Token 數
    *  **When** 應用程式準備發送請求
    *  **Then** 應用程式應只選擇相關性最高的片段傳送，並在必要時對其進行壓縮，以確保 Token 總數不會超過限制，避免 API 呼叫失敗。
